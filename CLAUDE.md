## File naming

All filenames: `snake_case`, no spaces, no uppercase.

| Category | Location | Pattern | Example |
|---|---|---|---|
| Raw dataset | `data/raw/` | `{dataset}_{split}_v{N}.{ext}` | `census_train_v1.csv` |
| Processed dataset | `data/processed/` | `{dataset}_{stage}_v{N}.{ext}` | `census_encoded_v2.parquet` |
| Figure | `outputs/figures/` | `{experiment}_{descriptor}.{ext}` | `baseline_confusion_matrix.png` |
| Model artefact | `outputs/models/` | `{model}_{experiment}_v{N}.{ext}` | `rf_baseline_v1.pkl` |
| Script | `src/` | `{NN}_{verb}_{descriptor}.py` | `01_preprocess_census.py` |
| Config | `config/` | `{component}.yaml` | `model.yaml` |
| Document | `docs/` | `{YYYYMMDD}_{descriptor}.{ext}` | `20260423_experiment_notes.md` |

Rules:
- `{NN}` is two-digit zero-padded (`01`, `02`, …). Unordered helpers use `util_` prefix instead.
- `{split}` ∈ `{train, val, test, full}`.
- `{stage}` ∈ `{cleaned, encoded, scaled, merged}`.
- `{model}` ∈ `{lr, rf, xgb, nn}`.
- Version `v{N}` is an integer starting at 1. Bump when inputs or parameters change.
- Never create files outside these directories without asking.

---

## Configuration files

- Format: **YAML only**. Never use JSON or TOML for config.
- All YAML keys: `snake_case`.
- Every config file must start with `# version: N`.
- Secrets never in YAML — use `.env` and environment variables.
- When editing a config value that affects reproducibility, increment the `# version:` number.
- Three config files exist; add new ones only for genuinely separate concerns:
  - `config/data.yaml` — paths, splits, preprocessing
  - `config/model.yaml` — algorithm, hyperparameters, artifact path
  - `config/eval.yaml` — metrics, thresholds, output paths

---

## Directory rules

- `data/raw/` — read-only after deposit. Never modify or overwrite raw files.
- `data/processed/` — derived data only. Always traceable back to a raw source + script.
- `outputs/` — generated by scripts. Safe to regenerate; never hand-edit.
- `src/` — pipeline scripts only. Exploratory notebooks go in `docs/` or a dedicated `notebooks/` dir.
- Do not create new top-level directories without asking.
